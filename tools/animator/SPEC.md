# Stickman Animation Editor — Product Design Spec

> 状态：草稿（2026-06）  
> 作者：根据代码逆向整理，存在不确定项（见文末 §9）

---

## 1. 背景与目标

### 为什么需要这个工具

Notebook Wars 使用一个程序化骨骼角色（stickman）作为战斗单位。角色由 11 根骨骼组成，通过关键帧动画驱动。游戏引擎需要在运行时对动画进行插值播放。

这个工具解决的问题：
- 没有合适的轻量外部工具（Spine 太重，DragonBones 导出格式不可控）
- 需要完全控制导出的 JSON 格式，以便游戏引擎直接消费
- 需要能预览 sprite 绑定效果（骨骼 + 贴图）

### 设计目标

1. **轻量**：单 HTML 工具，本地 DevServer，无服务端
2. **格式自主**：导出 JSON 完全由自己定义，游戏引擎运行时直接读取
3. **可迭代**：骨骼定义和 UI 都在代码里，改起来快
4. **够用即可**：不追求 Spine 级别的功能，只做游戏实际需要的

---

## 2. 用户与使用场景

**主要用户**：游戏开发者（当前仅 Tao）  
**使用频率**：制作或调整角色动画时

### 典型工作流

```
1. 打开工具（npm run dev）
2. 选择或新建一个动画片段（如 "walk"）
3. 拖动骨骼调整姿态
4. 在时间轴打关键帧（K 键 或 点 + Keyframe）
5. 播放预览，调整
6. （可选）导入图集，绑定 sprite 预览效果
7. 导出 animation.animator.json
8. 游戏引擎读取 JSON，运行时播放
```

## 3. 功能规格

### 3.1 骨骼系统

- **固定骨骼**：11 根，不可在工具内增删（修改需改代码）
  - `root`（髋部，不可选，仅作 FK 锚点）
  - `spine`、`head`
  - `r/l_upper_arm`、`r/l_lower_arm`（共 4 根）
  - `r/l_upper_leg`、`r/l_lower_leg`（共 4 根）
- **选择**：点击骨骼高亮，右侧面板显示属性
- **旋转**：左键拖拽，相对 pivot 点计算角度 delta
- **Pan**：右键拖拽画布

### 3.2 动画片段（Clip）管理

| 操作 | 入口 |
|---|---|
| 新建 | 左侧面板 "+ New" |
| 重命名 | "✎" 按钮 |
| 删除 | "🗑" 按钮（确认对话框） |
| 切换 | 点击列表项 |
| 加载预设 | 底部 "📋 Preset…"（idle/walk/attack/hurt/death/spawn） |

每个 clip 有独立的 `duration`、`loop`、`keyframes`。切换 clip 时工具栏的 Duration / Loop 输入框自动同步。

### 3.3 关键帧操作

| 操作 | 快捷键 / 入口 |
|---|---|
| 添加关键帧（当前时间、当前姿态） | `K` 键 或 工具栏 "+ Keyframe" |
| 删除关键帧 | `Delete` / `Backspace` 或 "✕ Del KF" |
| 跳转到前/后关键帧 | "⏮" / "⏭" 按钮 |
| 拖动关键帧（改时间） | 时间轴上左键拖拽菱形 |
| 设置 easing | 时间轴上右键菱形 → 上下文菜单 |
| 复制/粘贴关键帧 | 时间轴右键菜单 |

**关键帧是稀疏的**：一个关键帧只需包含该时刻有变化的骨骼，其他骨骼自动从相邻帧插值。

### 3.4 时间轴

- 显示所有可动骨骼的行（共 10 行：spine + head + 4 臂 + 4 腿）
- 关键帧以菱形显示，颜色表示属性类型：
  - 灰色：仅旋转
  - 白色：sprite 帧切换
  - 橙色：位移（translateX/Y）
  - 蓝色：缩放
- 选中关键帧高亮为青色
- 红色竖线为播放头，可拖拽 scrub

### 3.5 时长管理

- 手动：工具栏 Duration 输入框
- 自动：点 "Auto" 按钮，自动设为最后一个关键帧的时间

### 3.6 播放控制

| 控制 | 入口 |
|---|---|
| 播放 / 暂停 | `Space` 键、"▶ Play" 按钮 |
| 停止（回 0） | "⏹ Stop" 按钮 |
| 速度 | 0.25× / 0.5× / 1× / 2× |
| 循环 | Loop 复选框 |

### 3.7 Sprite 绑定

骨骼可以绑定图集帧，用于预览实际美术效果。

**流程**：
1. 在 Atlas 面板导入图集（TexturePacker JSON + PNG；PNG 可选，若 JSON 中 `meta.image` 是 data URL 则自动使用）
2. 在画布选中骨骼
3. 右侧 "Bone Properties" → Sprite Binding → 选择帧 → "Bind Sprite"（自动切换到 Sprite 预览模式）
4. 可调整 Anchor X/Y、Flip X

**预览模式**：`Tab` 键切换 Skeleton（线框）/ Sprite（贴图）。

### 3.8 挂点（Attachment Points）

非动画的固定标记点，相对于角色 root（髋部）：

| 挂点 | 默认偏移 | 用途 |
|---|---|---|
| Shadow（🔵） | (0, +52) | 脚下地面阴影位置 |
| Hit（✦） | (0, -80) | 受击特效播放点 |

在右侧面板 "Attachment Points" 区块编辑 offsetX/Y。  
渲染：shadow 显示为蓝色椭圆，hit 显示为红色准星。  
随项目 JSON 导出/导入。

### 3.9 辅助视图

| 选项 | 默认 | 说明 |
|---|---|---|
| Show joints | 开 | 显示关节圆圈 |
| Onion skin | 关 | 相邻关键帧半透明叠加 |
| Guide lines | 关 | 中心垂直参考线 |

### 3.10 Undo / Redo

- `Ctrl+Z` / `Ctrl+Shift+Z` / `Ctrl+Y`
- 覆盖所有数据操作（旋转骨骼、增删关键帧、sprite 绑定等）
- 最多 100 步

### 3.11 导出 / 导入

- 导出：`animation.animator.json`（包含 animations + bindings + attachmentPoints）
- 导入：读取同格式 JSON，合并进当前会话

## 5. 导出格式与游戏引擎对接

### 格式概要

```json
{
  "version": 1,
  "bindings": { "boneId": { "frameId": "...", "anchorX": 0.5, "anchorY": 0.5, "flipX": false } },
  "animations": {
    "walk": {
      "duration": 0.5,
      "loop": true,
      "keyframes": [
        { "time": 0.0, "bones": { "spine": { "rotation": 5 } } }
      ]
    }
  },
  "attachmentPoints": [
    { "id": "shadow", "label": "...", "offsetX": 0, "offsetY": 52 },
    { "id": "hit",    "label": "...", "offsetX": 0, "offsetY": -80 }
  ]
}
```

### 游戏引擎运行时需实现

1. 读取 JSON，建立 `AnimationStore`
2. 每帧调用 `sampleClip(clip, t)` 获取插值结果（`interpolate.ts` 可直接复用）
3. 用 FK 结果定位 sprite / 骨骼线框
4. 用 `attachmentPoints` 定位阴影、特效等

## 7. 预设动画

内置 6 个预设，可通过 "📋 Preset…" 加载：

| 名称 | 时长 | Loop | 说明 |
|---|---|---|---|
| idle | 1.5s | ✓ | 轻微呼吸摆动 |
| walk | 0.5s | ✓ | 走路循环 |
| attack | 0.6s | ✗ | 右手攻击 |
| hurt | 0.4s | ✗ | 受击后退 |
| death | 0.8s | ✗ | 倒地死亡 |
| spawn | 0.35s | ✗ | 出场跳入 |

## 8. Easing 类型

| 类型 | 描述 |
|---|---|
| `linear` | 匀速（默认） |
| `ease-in` | 慢入快出 |
| `ease-out` | 快入慢出 |
| `ease-in-out` | 慢入慢出 |
| `step` | 瞬间跳变（用于 sprite 帧切换） |

## 9. 待确认问题

以下内容不确定，需要讨论：

### 功能边界
1. **挂点是否需要更多？** 目前只有 shadow 和 hit，是否还需要武器挂点、特效挂点（如脚步烟尘）等？
2. **挂点是否应支持绑定到特定骨骼？** 现在是相对 root，如果需要"武器跟随手部"则需要 parent bone 概念。
3. **图集命名约定**：frame ID 的命名有没有约定？游戏引擎如何查找对应帧？

### 游戏引擎对接
4. **`interpolate.ts` 是否已被游戏引擎复用？** 还是游戏侧有自己的动画播放器？
5. **Sprite bindings 游戏侧如何处理？** 工具里 binding 是编辑器状态，游戏侧是否有对应运行时概念？
6. **`attachmentPoints` 游戏引擎读取方式**：目前只有编辑器端写入，游戏端代码在哪里？

### 工具本身
7. **骨骼数量/结构是否还会变？** 目前 11 根固定，是否有计划加手指、尾巴等？
8. **多角色支持的计划**？目前只有一个 stickman 骨架。
9. **Shadow 偏移 +52 / Hit 偏移 -80 的依据**？是否匹配当前骨骼尺寸，还是需要校准？
