# 美术设计总纲

版本：v0.1  
状态：草稿，待评审

---

## 一、核心视觉概念

### "一场发生在用过的笔记本上的战争"

本游戏的全部视觉语言围绕同一个核心隐喻展开：**这场战争是某个高中生在课堂上随手画出来的**。

这不是一张精心设计的战棋盘，也不是史诗级的奇幻战场——它是一个无聊下午的产物，画在方格纸上，用了几种不同颜色的笔，中间涂改过几处，边角还有几个跑题的涂鸦。

这个概念有三层含义：

**视觉上**：美术风格统一在"手绘笔记本"这一物理载体下，所有元素——地图、单位、UI、特效——都应该能让人相信"这是一个真实存在的人用笔画出来的"。

**情感上**：传达一种课间的轻松感、课堂上的叛逆感、和朋友互传本子的分享欲。不严肃，不宏大，是属于年轻人的游戏。

**差异化上**：当前手游市场充斥着精细3D和过度设计的卡通风，这套视觉在货架上具有天然的辨识度。

---

## 二、目标受众与文化定位

**核心受众：13～22岁，在校学生为主**

这个年龄段的用户正处于建立个人审美认同的阶段，对"属于自己这一代"的视觉风格有强烈共鸣。方格纸和涂鸦是他们每天接触的物理现实，用这个语言做游戏，能绕过"我为什么要玩这个"的心理门槛——他们会觉得"这就是我会做的事"。

**文化参照**：
- 课桌上涂鸦的少年感
- Stick War 系列的直白趣味性
- 涂鸦艺术（Doodle Art）的随性能量
- 学生笔记本里那些认真画出来的"无聊小人"

## 三、色彩系统

整套配色模拟**真实文具**的颜色，而非数字设计的色板。

### 3.1 基底色（背景层）

| 用途 | 颜色描述 | 参考值 |
|---|---|---|
| 方格纸底色 | 泛黄的米白，非纯白 | `#F5F0E8` |
| 格线颜色 | 浅蓝灰，低对比度 | `#C8D8E8` 透明度40% |
| 纸张暗角 | 边缘轻微压暗，模拟真实纸张 | — |

### 3.2 单位色（笔迹层）

每种单位对应一种真实文具颜色，用颜色区分兵种，不依赖形状细节：

| 单位 | 颜色 | 对应文具 |
|---|---|---|
| 普通兵（Swordsman） | 炭黑 / 铅笔灰 | 铅笔 / 黑色中性笔 |
| 盾兵（Guardian） | 深蓝 | 蓝色圆珠笔 |
| 弓箭兵（Archer） | 朱红 | 红色水笔 |
| 兵营（Barracks） | 墨绿 | 绿色马克笔 |
| 箭塔（Arrow Tower） | 深蓝（加粗线条） | 蓝色记号笔 |
| 敌方单位 | 同上各色，加橙红色外轮廓或阴影区分阵营 | — |

> **设计原则**：己方和敌方单位使用相同的兵种颜色，通过**描边颜色**或**背景标记**区分阵营，而不是靠换色（蓝方 vs 红方），以维持"都是同一个人画的"的统一感。

### 3.3 功能强调色

| 用途 | 颜色 | 对应文具 |
|---|---|---|
| 重要提示 / 警告 | 亮黄荧光 | 黄色荧光笔 |
| 选中 / 激活状态 | 橙色荧光 | 橙色荧光笔 |
| 不可用 / 禁用状态 | 灰色涂改液质感 | 修正液 |
| 法术 / 特效爆发 | 红色马克笔粗线 | 红色马克笔 |

### 3.4 禁止使用

- 渐变色（打破手绘感）
- 发光效果 / 描边辉光（与纸面质感矛盾）
- 超过5种主色同时出现在同一画面

## 四、单位美术规范

### 4.1 造型风格

所有单位采用**火柴人造型**，以下为具体规范：

- **线条**：模拟圆珠笔/中性笔笔触，线宽略有粗细变化，非完美等粗。禁止使用过于光滑的矢量线条。
- **比例**：头部略大（约占身高1/4），夸张动作时四肢可拉长，不追求写实比例。
- **细节上限**：每个单位的辨识特征不超过2个。普通兵——持剑；盾兵——圆盾遮住半身；弓箭兵——拉弓姿势。过多细节在小格子里无法被感知，只会增加制作成本。
- **阵营标记**：己方单位头顶小圆点（蓝），敌方单位头顶小叉（橙红），不改变单位本体颜色。

### 4.2 动画规范

动画帧数控制在**6～10帧**，保留手绘的跳跃感，不追求丝滑流畅——轻微的"卡顿感"反而符合涂鸦动画的美学。

| 动画类型 | 说明 |
|---|---|
| 行走 | 手脚交替，夸张摆幅，像被用力画出来的感觉 |
| 攻击 | 短促有力，击中瞬间画面轻微"抖动"（手绘颤动感） |
| 受击 | 身体短暂变形（挤压拉伸），非真实物理 |
| 死亡 | 倒地后变成一个叉，像被橡皮擦掉一样淡出 |
| 出生/放置 | 从一个点"画出来"，笔迹展开动画 |

### 4.3 建筑美术

建筑为**简笔几何形态**，像学生在草稿纸上随手画的房子、箭楼：

- 兵营：一个方形框加一面小旗，旗上画兵种图标
- 箭塔：梯形塔身加三角顶，顶部有一支弓
- 城墙：矩形砖块纹路（用斜线填充表示）
- 炮塔：圆形炮管从方形炮台伸出

所有建筑线条略粗于单位线条，以便在战场上作为"地标"被快速识别。

## 五、地图与背景规范

### 5.1 背景底层

- 方格纸纹理，格线颜色极浅，不喧宾夺主
- 纸张略有纹理感（非纯色填充）
- 边缘可有轻微泛黄和折痕，强化物理感

### 5.2 战场装饰（涂鸦层）

在地图的**非战斗区域边缘**点缀少量涂鸦元素，强化"用过的笔记本"质感：

- 边角偶尔出现小太阳、星星、箭头、感叹号等随手涂鸦
- 偶尔有一行被划掉的"草稿字"（可以是无意义符号）
- 基地区域旁可有类似"BOSS"或"[START]"的手写标注
- 所有装饰元素**只出现在格子外侧边缘**，不进入格子战斗区域，不干扰游戏信息

### 5.3 基地视觉

双方基地为**2×2格**的手绘城堡图标，简笔轮廓，内含HP血条（用铅笔格子填充的血量格）。血量减少时，城堡轮廓出现裂缝涂鸦。

## 六、UI规范

### 6.1 核心原则

> **装饰层可以"乱"，功能层必须清晰。**

手牌、金币数字、血条等核心信息需要在0.5秒内被玩家读取，这些元素不能被涂鸦感干扰。涂鸦感只用于背景装饰、卡片边框、按钮纹理等非信息载体的部分。

### 6.2 手牌设计

- 每张卡牌是一张**便签纸或小卡片**的形态，带轻微卷角或撕边效果
- 卡面内容：单位/建筑/法术的简笔图标（与游戏内造型一致）+ 费用数字（手写字体）
- 手牌排列在屏幕底部，轻微扇形展开
- 无法出牌时：卡牌覆盖一层"涂改液白"的半透明遮罩，费用数字被划掉

### 6.3 金币与资源显示

- 金币数字：手写数字字体，配一个简笔硬币图标
- 上限提示：数字旁出现一个小感叹号涂鸦（非弹窗）
- 资源加速时：数字旁出现速度线（手绘的三条斜线）

### 6.4 字体

- 数字与核心信息：圆润手写感等宽字体，清晰优先
- 标题/装饰文字：随意手写风字体，可略歪
- 禁止使用：系统默认字体、过度设计的艺术字体

### 6.5 按钮与菜单

- 按钮形态：矩形，带不规则手绘描边（非完美圆角矩形）
- 按下反馈：轻微下压 + 纸张褶皱动效
- 菜单背景：大张方格纸，带笔记本装订线（左侧红色竖线）

## 七、特效规范

特效是笔记本美学中最容易出戏的环节，需要严格约束：

| 特效类型 | 实现方式 |
|---|---|
| 普通攻击命中 | 手绘星形爆炸符号（漫画感），1～2帧出现后淡出 |
| 陨石打击法术 | 红色马克笔粗线从上方划下，落点出现涂鸦爆炸圈 |
| 急速冲锋法术 | 单位身后出现手绘速度线（3～4条平行线），持续5秒 |
| 单位死亡 | 身体变成一个手绘的"×"，0.3秒后淡出消失 |
| 建筑摧毁 | 建筑图标碎成几个线段，像被用力划掉 |
| 基地受击 | 基地图标抖动，出现新的裂缝涂鸦线条 |

**禁止**：粒子爆炸、光效、烟雾等数字游戏常见特效——这些在纸面世界里不存在。

---

## 八、声音方向（参考）

声音不属于美术范畴，但视觉风格对音效有强烈暗示，供参考：

- 整体音效应偏向**轻巧、卡通、非写实**，与视觉的手绘感匹配
- 可参考：铅笔划过纸张的沙沙声、橡皮擦声、翻笔记本的声音作为UI音效
- 禁止：金属碰撞、爆炸轰鸣等写实战争音效

## 十、美术范围（MVP阶段）

### 必须完成

- [ ] 3种单位的静态造型与动画（行走、攻击、受击、死亡）
- [ ] 2种建筑的静态造型（兵营、箭塔）
- [ ] 地图背景（方格纸底 + 基地图标）
- [ ] 手牌UI（卡牌外框、费用显示、选中/禁用状态）
- [ ] 核心特效（攻击命中、死亡、陨石打击、急速冲锋）
- [ ] 金币/血条等核心HUD元素

### MVP后扩展

- [ ] 更多单位与建筑的美术
- [ ] 对战前/结算界面的完整设计
- [ ] 笔记本封面风格的主菜单
- [ ] 过场动画（手绘翻页效果）
- [ ] 多套"笔记本主题"换肤（不同纸张颜色/笔迹颜色）
