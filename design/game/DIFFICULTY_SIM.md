# 关卡难度模拟器（PvE Balance Tool）

> 用真实确定性战斗引擎 + 一个基线玩家 AI，量化战役关卡难度。
> 代码：`client/test/difficultySim.ts`（核心）/ `difficulty.test.ts`（难度矩阵+回归守卫）/
> `experiment.test.ts`（关卡调参 A/B）/ `diag.test.ts`（单关逐秒时间线检视）。

## 这是什么

战役关卡此前**没有**任何难度评估手段，全靠手感。本工具用 `@nw/engine`（30Hz、
确定性）无渲染地跑一整场战役，由一个固定策略的「基线玩家 AI」自动防守，输出：

- 能否通关（survive：撑到波次耗尽且清场）
- 结束/最低基地血、撑了多少秒、首次破防时刻
- 峰值同屏敌人数 / 总血量

并提供两个上层能力：
- **养成门槛扫描** `findClearThreshold(levelId)`：fresh→T2…T6 逐档跑，找能过的最低养成。
- **关卡调参 A/B** `simulateLevel(levelObj, …)`：传入改过参数的关卡对象（startInk /
  inkRegenMult / enemyScale / 波次 atTick·count），对比下调方案效果。

### 跑法

```bash
cd client
npx vitest run difficulty    # 难度矩阵（ch1 各关 × 养成预设）+ 确定性/单调性守卫
npx vitest run experiment    # ch1_lv1 各下调方案对比
npx vitest run diag          # 单关逐秒时间线（改 diag.test.ts 顶部 LEVEL_ID/PRESET 换关）
```

## 基线 AI 策略（启发式，非最优）

每 tick 用令牌桶按 `actionsPerSecond`（默认 6，模拟人手 APM）连续出手，优先级：

1. **流星 AOE**：某车道聚 ≥2 敌人就砸（每 tick 一发）。
2. **防线覆盖**（最高战术优先）：任何来敌车道我方阻挡 < `blockersPerLane`（默认 2）
   就在最逼近的欠守车道补兵——**本游戏防御主力是野战单位，必须每条来敌道都有人守**。
3. **箭塔骨架**：覆盖满足后补塔（塔在 row0、射程 2，只是后排火力，非主力）。
4. 兵营（被动出兵流）。
5. 余力把多余墨继续分摊到防守最薄弱的来敌车道。
6. 经济：场面安全且有余钱才升基地（提升回墨）。

### ⚠️ 重要局限（解读结果必读）

基线 AI **明显弱于熟练玩家**（满养成下仍过不了大半 ch1，而游戏是能正常游玩的）。
因此：

- 工具给的是**相对难度**和**养成是否有用**，**不是**「最优解能否通关」。
- **绝对通关与否偏保守**：AI 能轻松过 ⇒ 关卡偏易；AI 勉强过/过不了 ⇒ 偏难。
- 调参 A/B 看的是**方向**（哪个杠杆有效、改动量级），不是精确数值。

> 改进 AI（走位/集火/流星时机/卡序）可提升其代表性，是后续可做项。

## ch1 难度矩阵（基线 AI，本次快照）

```
level   |fresh |T2  |T3  |T4  |T5  |T6  |min通关
ch1_lv1 | ✗    |✗   |✗   |✗   |✗   |✗   |过不了
ch1_lv2 | ✗    |✗   |✗   |✓18 |✓77 |✓100|T4
ch1_lv3 | ✗    |✗   |✗   |✗   |✗   |✗   |过不了
ch1_lv6 | ✗    |✗   |✗   |✓44 |✓3  |✓59 |T4
…其余 ch1 关：基线 AI 均未过（受 AI 水平所限，偏保守）
```

- **lv1 比 lv2/lv6 还难**（后两者 T4 即可过，lv1 连 T6 都过不了）——首关是个**错位的难度尖峰**。
- 数值非严格单调（如 lv6 T5 仅 3 血、T4 反而 44 血）反映 AI 是启发式、有噪声。

## ch1_lv1 诊断结论（关键）

引擎铁的事实（不依赖 AI 水平）：

- 敌人 **tick 60 = 2.0s** 就到，开局 0 墨、回墨恒定 **2/s**、基地升级 50/100/200——
  **没有经济起步窗口**。
- 波次：**34 个 Max**（190 血 / 22 攻 / 残血双倍伤）+ **2 个 Lena Boss**（150 血 / 每击 -8 减伤），
  在 2s–43s 内投放，含 **tick 900（30s）一次性 12 个 Max 齐推**。敌方总血量 ≈ **6,760**。

调参 A/B 验证（fresh + 基线 AI）：

| 方案 | 结果 |
|---|---|
| 原版 | ✗ 34s 阵亡 |
| 只加经济（起手墨/回墨×2~2.5） | ✗（30s 的 12 连推照样压垮） |
| 只砍兵量×0.5+推迟5s（峰值仅 9 敌） | ✗（fresh 经济仍太薄，养不起防线） |
| **经济 + 节奏一起松**（起手墨30/回墨×1.3/敌HP×0.85/推迟+兵量×0.55） | ✓ **满血 100 通关** |

**结论：lv1 的命门是「开局经济 vs 早期敌人数量」，不是单体战力。**
→ **加养成（战力）救不了第一关**（fresh 撑 34s、T6 也只撑 55s，全失败）；
→ 单给经济或单砍敌人都不够，**必须经济 + 波次节奏一起松**。

## lv1 推荐下调方向（经工具验证，具体数值可用 experiment.test.ts 自行 A/B）

1. **给开局经济缓冲**（最高杠杆）：`startInk` ≈ 25–30，或 `inkRegenMult` ≈ 1.3–1.5。
2. **软化开局与 30s 尖峰**：首波推迟到 ~5s；整体波次 count 砍 ~40–50%，尤其 tick 900 的 12-Max 齐推。
3. 可选：首关 `enemyScale.hp` ≈ 0.85。

> 三者组合（推荐B）让基线 AI 满血通关；考虑到 AI 偏保守，实际人类体验会更宽松，
> 适合作为第一关。最终手感建议用 `experiment.test.ts` 微调到「基线 AI 结束血 ~40–70」区间。
