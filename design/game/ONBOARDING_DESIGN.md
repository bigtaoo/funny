## 1. 设计目标

把"陌生玩家第一次打开 → 理解核心玩法 → 完成第一场胜利 → 有理由明天回来"压进**一条尽量短、可跳过、不劝退**的动线。北极星 = **首战完成率**与**D1 回访**。

**基调原则**（与全局一致）：温和、不强制、不打断手感——和 ADR-009/011 的「装备失败不碎、断签不惩罚」同一克制风格。新手引导**可跳过、可重看**，不锁玩家。

## 3. 战斗内教学覆盖层（coach marks）

第一场战斗里，用**非阻塞的轻提示**逐步教会核心操作，每步由玩家动作推进、随时可跳过：

| 步 | 触发时机 | 教什么 | 形式 |
|---|---|---|---|
| T1 | 进场 | 「这是你的笔记本（基地），别让它被画花」 | 高亮基地 + 一行提示 |
| T2 | 墨够出第一张牌 | 「点这里出兵」 | 高亮手牌 + 手指引导 |
| T3 | 首个敌人出现 | 「红色是老师的批改军，挡住它」 | 高亮来袭车道 |
| T4 | 首次出兵后 | 「墨滴会自己回涨，攒墨出更强的」 | 高亮墨条 |
| T5 | 撑过首波 | 「撑过所有波次就赢，加油」 | toast |

- **实现层**：覆盖层是**表现层 UI**（render/scene），订阅引擎事件触发，**不进 `client/src/game` 纯引擎**（同音频红线，保确定性/回放/裁判不受影响）。
- **进度持久化**：`SaveData.flags.tutorial_step`（或 `tutorial_done`），完成/跳过后不再弹；设置页可「重看教学」。
- **i18n**：`tutorial.*` 全语种（zh/en/de）。
- **可跳过**：右上角常驻 skip，跳过即 `tutorial_done=true`。

## 4. 功能渐进解锁（progressive disclosure）

新玩家一进大厅就铺满商店/盲盒/社交/SLG/排位会**信息过载**。按里程碑**逐步点亮**入口，未解锁的格子灰显 + 「通关第 N 关解锁」气泡：

| 功能 | 解锁条件（建议，数字待调） | 理由 |
|---|---|---|
| 战役 | 一开始就有 | 唯一正门，教学载体 |
| 商店 / 盲盒 | 首胜后 / 拿到第一笔金币 | 先懂玩法再谈消费 |
| 社交（好友/私聊） | 通 ch1 若干关 | 留存中段再社交 |
| 排位 PvP | 通战役第一章 / 达等级门槛 | 避免新手被虐劝退 |
| SLG 大世界 | 更后置（达更高进度） | 最重系统，最后开 |

- **门控数据**：解锁阈值集中一处常量（建议 `shared` 或客户端 `onboarding.ts` 单一来源），客户端据 `progress` 判定灰显/点亮。
- 与 SA-4「offline 模式社交/联机入口路由到登录」**叠加**：先过登录门，再过解锁门。
- **不硬锁付费**：商店解锁是"引导节奏"非"付费墙"，解锁后才显但不强迫。

## 5. 合规挂钩（与 COMPLIANCE_GLOBAL 对齐）

FTUE 是合规弹窗的落点，顺序很关键：

1. **① 年龄声明门**：首启、neutral age gate（不诱导），结果影响分级/COPPA 判定。见 [`COMPLIANCE_GLOBAL.md`](COMPLIANCE_GLOBAL.md) §3.4。
2. **隐私政策入口**：登录/设置页可点（COMPLIANCE §3.1）。
3. **EU/UK 采集同意弹窗**：按地区粗判，**在开始埋点采集前**弹（COMPLIANCE §3.3）——即应排在 IntroScene 埋点之前。
4. 年龄门 + 同意状态持久化进 `flags`，不重复弹。

> 顺序铁律：**年龄门/同意 → 才开始埋点采集**。否则首启漏斗事件本身就违规。

---

## 6. 留存 / 回访钩子（与 RETENTION 对齐）

FTUE 末端要埋下"明天回来"的理由，但**不新增金币龙头**（ADR-011）：

- 首胜即时奖励（一次性，软通货为主）。
- 首胜后**引出每日签到/每日任务**入口（机制权威归 [`RETENTION_DESIGN.md`](RETENTION_DESIGN.md)），让玩家看到"明天有东西拿"。
- D0 结束提示下一关/明日签到，不强推。

---

## 7. 漏斗埋点（字段权威归 ANALYTICS）

FTUE 每个关键节点打点，用于诊断流失。**事件字段定义归 [`ANALYTICS_DESIGN.md`](ANALYTICS_DESIGN.md)**，本文只列**该埋哪些节点**：

`首启 → 年龄门通过 → 同意（如适用）→ intro 完成/跳过 → 登录方式（试玩/匿名/正式）→ 首战开始 → 教学各步完成/跳过 → 首战结果（胜/负/退出）→ 首胜领奖 → 次日回访`。

> 采集受 §5 同意门控；EU/UK 未同意则不采上述行为事件。
