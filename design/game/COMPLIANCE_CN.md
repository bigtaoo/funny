## 1. 范围与定位

- **覆盖**：中国大陆（iOS 中国区 / 安卓各应用商店 / 微信小游戏）上架与商业化运营。
- **不覆盖**：海外渠道（隐私政策 / 年龄分级 / GDPR / 平台 IAP）——见 [`COMPLIANCE_GLOBAL.md`](COMPLIANCE_GLOBAL.md)。
- **一句话原则**：**没有版号就不能在中国商业化运营**；拿到版号后，**实名 + 未成年人防沉迷（限时限额）+ PIPL 个人信息保护**是三道必须在代码里落地的硬门，其中**未成年人限时**与**充值限额**最容易在版署抽查 / 玩家投诉时翻车。

## 3. 版号（前置一切）

- **没有版号 = 不能在中国大陆商业化运营**（不能内购、不能上架正式版）。版号经**出版社**向**国家新闻出版署**申报，周期长、需游戏接近完成形态（含防沉迷/实名落地的版本）。
- **依赖发行/出版方**：版号申报通常由有资质的发行商/出版社代办；本文工程侧需配合提供：可玩的送审包（含实名+防沉迷+充值限额已落地）、游戏说明、内容材料。
- **与海外并行**：海外测试先行（ADR-013），同时启动版号申报——两条线不互相阻断。**本文 §4–§7 的工程项在送审包里必须已实现**（送审版即需带防沉迷/实名）。

## 4. 实名认证（防沉迷的前置）

- **要求**：玩家须经**真实身份信息**认证后方可进入游戏（对接版署「网络游戏防沉迷实名认证系统」或合规第三方认证服务）。游客/试玩模式受**严格限制**（如累计时长后强制实名，具体以规定为准）。
- **落地（account 扩展）**：
  - 注册/首次进入加**实名认证门**：采集姓名 + 身份证号 → 调认证接口校验 → 账号标记 `realNameVerified` + 年龄段（由身份证推算 `ageBand`，**只存判定结果/必要字段，明文身份信息按 PIPL 最小化与加密**）。
  - `ageBand` 决定防沉迷（§5）与充值限额（§6）档位，由服务器权威持有，客户端不可改。
  - 一人一号映射、防小号绕过（同身份证关联，规定要求）。
- **与海外账号体系的隔离**：海外 `deviceId` 匿名账号底座（GLOBAL §3.2）在中国区**不足以满足实名**——中国区必须实名后才解锁完整功能。两套登录策略按区构建/配置分流，不混用。

## 5. 未成年人防沉迷（限时，最硬的一条）

> 现行规定（2021）：网络游戏企业仅可在**周五、周六、周日及法定节假日的 20:00–21:00** 向未成年人提供 1 小时服务，其余时间一律不得。阈值以届时最新规定为准。

- **判定主体**：服务器（按实名 `ageBand` 判未成年人；时段/时长闸**服务器时钟权威**，客户端改本地时间无效，同 RETENTION dayKey / S1 节拍器思路）。
- **时段闸**：未成年人账号仅在允许时段可进入对局/养成核心玩法；非允许时段登录后拦截，提示「未成年人保护时段限制」。
- **时长闸**：允许时段内累计游戏时长达上限（当前 1h/日）即强制下线，期间到点提醒。
- **落地挂点**：
  - meta 鉴权 / 进入游戏的服务器入口校验 `ageBand + 服务器时间窗`；
  - gameserver 开局前由 ticket / meta 校验通过才放行（未成年人非允许时段不签 ticket / 拒开局）；
  - 时长计数按 `dayKey` 服务器侧累计（同 `victoryDaily`/`adsDaily` 计数范式）。
- **成年人不受限**：`ageBand=adult` 跳过时段/时长闸（但仍受充值理性提示等一般规定）。
- **公示**：设置页/启动页展示防沉迷规则与「健康游戏忠告」。

## 6. 未成年人充值限额（分龄，commercial 落地）

> 现行分龄限额（2019，阈值以最新规定为准）：

| 年龄段 | 单次充值上限 | 每月累计上限 |
|---|---|---|
| **< 8 岁** | **不得充值**（一律拒绝） | 0 |
| **8–16 岁** | ≤ 50 元/次 | ≤ 200 元/月 |
| **16–18 岁** | ≤ 100 元/次 | ≤ 400 元/月 |
| **成年人** | 无此限（仍受一般消费者保护） | — |

- **落地（commercial 扩展）**：充值/下单前服务器按实名 `ageBand` 校验单次额度 + 当月累计额度（`monthKey` 原子计数，类比 dayKey），超限拒绝并提示；`<8` 直接拒付。**额度判定服务器权威**，客户端只展示。
- **与海外内购的区别**：中国区充值走**国内合规支付渠道**（应用商店各自的支付 + 微信支付等），不是 Apple/Google IAP（海外 GLOBAL §5）。`recharge/verify`（SERVER_API §9）的平台验签实现按区分流。
- **金币经济不变**：限额是**充值闸**，不改 §6.1 金币产消模型；F2P 仍可正常玩（养成只让 PvE/SLG 体验更好，不碰公平 PvP，ADR-009）。

## 7. PIPL / 个人信息保护

- **同意**：采集个人信息（实名身份、设备、行为埋点）需**明示告知 + 取得同意**，敏感个人信息（身份证号）单独同意。复用 analyticsvc 顶层 `enabled` 开关（ANALYTICS §10）作埋点撤回闸门，并补**首次同意弹窗 + 隐私政策（中国区版本）**。
- **境内存储**：在中国境内运营收集的个人信息**原则上存储在境内**——后端 Mongo/Redis 需有**境内机房部署**；如需出境（如跨区账号），走 PIPL 出境安全评估/标准合同（依规模与数据类型）。**部署架构待定（§10 待办）**。
- **数据权利与删除**：账号删除 / 数据导出能力复用 `POST /account/delete`（SERVER_API §2.10）+ analyticsvc 按 `user_id` 批删；中国区删除范围与留存义务（如交易记录依法留存最小集）按法务定。
- **最小化与加密**：实名明文身份信息按最小化原则采集、加密存储，仅保留判定所需（`ageBand`/`realNameVerified` 标记 + 必要校验留痕）。
- **第三方共享**：广告/分析 SDK 的数据共享需在隐私政策（中国区）如实披露并取得同意。

## 8. 内容审查（送审自查）

- 题材（笔记本 / 文具 / 校园涂鸦对战）天然低敏感，但送审仍需自查：**无真实赌博呈现**（盲盒是付费随机道具、须概率公示，非赌博玩法，措辞/表现避免赌博暗示）、无血腥暴力（卡通涂鸦风，阵营色我蓝敌红为视觉对立非血腥）、无政治/宗教/历史敏感、社交内容（昵称/私聊）有治理（敏感词 + 举报，复用 GLOBAL §7 同一套过滤）。
- 叙事铁律（ADR-008，陶=中国/Anna=德国「东西碰撞」）涉及国别表达，送审前由发行/法务核内容口径，避免敏感表述。

## 10. 待办（开发顺序，跟版号节奏）

1. **确认发行/出版方 + 启动版号申报**（前置一切；决定送审版本形态）。
2. **实名认证门 + `ageBand` 权威字段**（防沉迷/限额的前置，送审版必备）。
3. **未成年人防沉迷限时**（时段 + 时长，服务器权威，送审版必备）。
4. **分龄充值限额**（commercial 按 `ageBand` 封顶 + `<8` 拒付，送审版必备）。
5. **PIPL：中国区隐私政策 + 首次同意弹窗 + 敏感信息单独同意**。
6. **数据境内存储部署架构**（境内机房；出境评估如需）。
7. **适龄提示 / 健康游戏忠告 / 抽卡概率公示页**（公示类，与海外公示页共用数据源）。
8. **国内合规支付渠道接入**（替换 dev 桩，按区分流）。
9. **内容自查 + 送审材料配合**（依发行方要求）。
