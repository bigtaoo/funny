## 一、屏幕规格与缩放模型

### 1.1 双设计空间

游戏支持竖屏和横屏两套完整布局，各自有独立的设计分辨率：

| 方向 | 设计分辨率 | 格子尺寸 | 战场（col×row） |
|---|---|---|---|
| 竖屏（Portrait） | 1080 × 1920 | 84 × 84 px | 12列 × 18行 = 1008 × 1512 px |
| 横屏（Landscape） | 1920 × 1080 | 70 × 70 px | 18列 × 12行 = 1260 × 840 px |

> 逻辑层坐标系永远是竖屏定义（col 0-11，row 0-17），横屏布局由渲染层做坐标映射。

### 1.2 缩放模型

所有元素尺寸以设计分辨率为基准，运行时做以下两层缩放：

**游戏层（Contain 缩放）**
```
gameScale = min(screenW / designW, screenH / designH)
```
游戏内容完整显示在屏幕内，多余空间留白。游戏容器居中。

**背景层（Cover 缩放）**
```
bgScale = max(screenW / designW, screenH / designH)
```
背景图（笔记本纸张纹理）始终铺满整个屏幕，超出部分裁剪。

两层作为独立 PIXI Container，游戏层内部代码全部使用设计坐标，无需额外换算。

### 1.3 方向检测与切换规则

- 检测方式：`window.innerWidth > window.innerHeight` → 横屏，否则竖屏
- **允许切换**：玩家在大厅时，监听 `resize` 事件实时切换方向并重建场景
- **锁定切换**：离开大厅后（进入匹配/游戏/结算）停止监听，方向不再变化

## 二、整体布局

### 2.1 竖屏布局（设计空间 1080 × 1920）

```
┌─────────────────────────────┐  ← 1080px
│  顶部 HUD                    │  70px   计时器 / 敌方HP / 设置
├─────────────────────────────┤
│                             │
│         战  场               │  1512px  12列 × 18行，cellSize=84
│   (含基地、建筑行、战斗区)    │         左右各 36px 边距
│                             │
├─────────────────────────────┤
│  底部资源 HUD                │  70px   金币 / 己方HP / 升级按钮
├─────────────────────────────┤
│  手牌区                      │  268px  6 张手牌
└─────────────────────────────┘
```

总计：70 + 1512 + 70 + 268 = 1920 px

### 2.2 横屏布局（设计空间 1920 × 1080）

逻辑坐标映射：col → 屏幕 Y，row → 屏幕 X。己方（玩家0）基地在左侧，敌方在右侧。

```
┌──────────────────────────────────────────────┐  ← 1920px
│  [ 计时器 ]    [ 敌方 HP 条 ]    [ ⚙ 设置 ]  │  60px  顶部 HUD
├───────────────┬──────────────────┬────────────┤
│               │                  │            │
│  (左边距)     │     战  场        │  (右边距)  │  840px
│   ~330px      │  1260 × 840 px   │   ~330px   │  cellSize=70
│               │  (18列 × 12行)   │            │
├───────────────┴──────────────────┴────────────┤
│ [金币+HP条] [卡1 卡2 卡3 卡4 卡5 卡6] [升级]  │  180px  底部条
└──────────────────────────────────────────────┘
      ~300px          ~1420px            ~200px
```

总计（高）：60 + 840 + 180 = 1080 px  
战场居中，左右各留约 330px 由背景图填充。

### 2.3 玩家视角镜像（联机预留）

| 玩家 | 竖屏 | 横屏 |
|---|---|---|
| 玩家0（本地） | 己方在下，面朝上 | 己方在左，面朝右 |
| 玩家1（本地） | 己方在下，面朝上（180° 旋转视图） | 己方在左，面朝右（180° 旋转视图） |

> **镜像路径现已激活**：PvP 匹配（含 botsvc）会把真实加入方分配为玩家1（`Side.Top`），
> 走 `createLayout(..., Side.Top)` 的 180° 镜像视图。此前该路径长期休眠，`LandscapeLayout`/
> `PortraitLayout` 的 `playerBaseRect()`/`enemyBaseRect()` 在 Top 分支把己方/敌方基座贴图放反了
> ——与 `gridToScreen`（兵种、出兵、战斗、裂纹/受击特效都走它）相反。表现为：**加入方打对方城堡，
> 受击特效却画在自己基地上**。已修正：两个 layout 的 Top 分支现在与 `gridToScreen` 对齐
> （单测锁定 `基座贴图中心 == gridToScreen(己方/敌方物理中心)`）。

## 三、手牌区

- 6 张卡牌横向排列，均匀分布
- 卡牌尺寸：约 155 × 190 px（宽 = (1080 - 间距) / 6）
- 卡面内容：兵种/建筑/法术简笔图标 + 费用数字（手写字体）
- 状态：
  - 正常：正常显示
  - 选中：卡牌上移约 20 px，高亮描边
  - 禁用（金币不足）：涂改液白半透明遮罩，费用数字被划掉
  - 即将刷新：半透明橡皮擦纹理从底部向上覆盖，卡面内容始终可见（透过遮罩可读）

### 手牌刷新机制

- 打出一张牌 → 该槽立即随机抽新牌，新牌计时器从 2 分钟重新开始
- 2 分钟未使用 → 该槽自动随机刷新为新牌
- 牌池无限循环，不会用完
- 开局时 6 张牌的计时器随机错开（各自 0～60 秒随机偏移），防止多张同时刷新

### 刷新倒计时视觉

橡皮擦效果：米白色半透明纹理从卡牌底部向上缓慢覆盖，2 分钟走完全程。
- 半透明（约 60% 不透明度），卡面图标和费用始终可读
- 覆盖完成时短暂停顿，卡牌"重新画出"新牌（出生弹出动效）
- 刷新时不打断玩家操作，静默替换

## 四、出牌交互

所有出牌均采用**选牌 → 拖拽**手势，三类卡牌反馈机制不同，手势一致。

### 兵种卡

1. 点击手牌 → 10 条进攻路线全部高亮（绿色=可放，灰/红=出兵行已占）
2. 拖拽至目标列任意位置（只识别列，不识别行）→ 该列格子放大提示
3. 出兵行被占 → 该列变红，禁止放置
4. 松手 → 单位在出兵行（row 16）生成，开始推进，扣除金币
5. 拖出战场 → 取消

> 单位永远从出兵行出发，玩家无法指定行。

### 建筑卡

1. 点击手牌 → 己方建筑行中未放置建筑的格子显示暗色预览图标
2. 拖拽至目标格 → 预览图标变亮
3. 已有建筑的格子无提示，不可放
4. 松手 → 建筑生成，扣除金币
5. 拖出战场 → 取消

### 法术卡

1. 点击手牌 → 卡片高亮
2. 拖入战场 → 影响范围高亮（吸附最近格子中心）
   - 陨石打击：2×2 四格高亮
   - 急速冲锋：无需选格，松手立即对己方全体单位生效
3. 松手 → 法术立即生效，扣除金币
4. 拖出战场 → 取消

### 通用规则

- 金币不足：卡牌为禁用状态，不可拖拽
- 拖回手牌区或拖出战场边界：均视为取消，卡牌归位

## 五、HUD 信息层级

### 5.1 竖屏 HUD

**顶部 HUD（70px）**

```
[ 计时器 ]          [ 敌方基地 HP 条 ]          [ 设置按钮 ]
  左侧                  居中（对准基地）              右侧
```

**底部资源 HUD（70px）**

```
[ 金币图标 + 数字 ]   [ 己方基地 HP 条 ]   [ 升级基地按钮 + 费用 ]
     左侧                  居中                      右侧
```

- **升级按钮**：拖拽到己方基地区域松手触发（与出牌手势一致），不可拖动时灰显

### 5.2 横屏 HUD

**顶部 HUD（60px）**

```
[ 计时器 ]          [ 敌方基地 HP 条 ]          [ 设置按钮 ]
  左侧                  居中                        右侧
```

**底部条（180px）—— 从左到右**

```
[ 金币 + 己方HP条 ]    [ 卡1  卡2  卡3  卡4  卡5  卡6 ]    [ 升级按钮 ]
      ~300px                      ~1420px                      ~200px
```

- 手牌卡片在横屏下缩小至约 200 × 160 px（`cardWidth=200, cardHeight=160, margin=8`），6张总宽 1240px（含间距），居中于 1420px 手牌区
- 左侧区域：金币图标+数字（上），己方基地 HP 条（下）
- 右侧区域：升级基地按钮（拖拽触发，同竖屏）

### 5.3 通用 HUD 规则

- 不显示敌方金币、不显示阶段倍率数字
- 阶段变化通过基地 buff 特效表现（产金加速：金币飞出动效；攻击×2：单位红色描边）

### HP 条设计（笔记本风格）

- 形式：方格填充血条，共 10 格，每格 = 10% HP
- 满血：格子全部填充实心
- 掉血：对应格子变为灰色空格
- 危险（≤30%）：剩余格子变红

### 阶段变化提示

不在 HUD 显示倍率数字，改为基地 buff 特效：
- 产金加速（×1.5 / ×2.0 / ×4.0）：基地旁出现速度线 + 金币飞出动效
- 攻击倍率（13min，×2）：所有单位出现红色描边特效

## 六、战场单位血条

- **单位**：受击时显示血条，3 秒后淡出；未受击时不显示，保持战场清晰
- **建筑**：血条常驻显示（建筑是战略目标，玩家需要持续判断）
- 血条风格沿用笔记本美术：细长方格填充条，掉血格子变为灰色空格

## 七、对战匹配流程

- 所有对战统一走匹配流程，无单独"vs AI"入口
- 无可用真人玩家时，默认匹配机器人；机器人有随机名字、等级等信息，与真人外观一致
- 卡牌详情（属性、说明）**仅在大厅**可查看，游戏内不提供，避免打断操作节奏

## 八、结算界面

### 胜负显示

- 大标题显示胜/负，手写风格字体，带简单动效（胜利纸张飞溅，失败打叉）

### 表现评级徽章

双方玩家各获得一枚或多枚徽章，无论胜负。徽章为手绘风格图标 + 称号文字。

| 称号 | 判定依据 |
|---|---|
| 最佳输出 | 对敌方基地造成伤害最高 |
| 铁壁防线 | 己方基地受到伤害最少 |
| 兵海战术 | 单局派兵数量最多 |
| 建筑大师 | 建筑存活时间总和最长 |
| 精准打击 | 法术命中单位数量最多 |
| 以少胜多 | 用较少金币消灭较多敌方单位 |

### 己方最大成就

可获得多枚徽章，但大字突出显示**得分最高的那枚**。例如同时获得"最佳输出"和"精准打击"，则突出显示"最佳输出"。结算画面用醒目手写大字呈现，例如：

> **「你消灭了 47 个敌方单位」**  
> **「你的基地只受到了 120 点伤害」**

文字样式：大号手写字体，带下划线或荧光笔标注效果，强化"笔记本批注"感。

- **内容栈垂直居中**于 header 与底部导航之间，自适应横竖屏，消除中间留白。

- 方向适配：同游戏内缩放模型（Contain + Cover），大厅支持方向实时切换。

#### 离线模式（SA-4）

离线（无账号）时隐藏所有需要账号的入口，不再灰显占位：

- **隐藏**：大世界 pillar、底部导航的商店 / 社交槽（导航变 3 槽：卡组 / 战绩 / 主页，等分铺满）、每日 / 限时活动行（仅在线下发）。
- **保留**：战役（本地 PvE，大世界隐藏时占满整行）、卡组 / 战绩（读本地存档）。
- **Hero 改为「人机对战」**：离线点击走本地 AI 对局（onStartGame），按钮文案 `人机对战`、副标题 `单局 5-10 分钟`。
- Header 右侧改为「登录/注册」入口（无金币/段位）。

### 匹配流程

**匹配中**：不跳转新页面，原地更新开始匹配按钮状态——按钮变为"匹配中…"，带动效（如笔画循环描边或跳动省略号），点击可取消。

**匹配成功**：进入 A vs B 界面：
- 全屏展示双方头像 / 名字 / 等级，中间一个手绘风格"VS"
- 此界面同时作为游戏资源加载画面，加载完成后自动进入游戏
- 展示时长约 2～3 秒（加载时间视情况延长）
